If you're integrating an AI agent with a service that performs real-world
actions — charging cards, provisioning seats, calling APIs, writing files —
OKORO is the verification layer between the two.

<SdkVersionBadges />

## Five-minute path

1. **Install** the SDK for your language — see badges above.
2. **Mint a keypair** with `generateKeypair()`. Private key stays in your process.
3. **Register the agent** via `okoro.agents.register({ publicKey })`. OKORO only ever sees the public key.
4. **Issue a scoped policy** with `okoro.policies.create({ scope, ttlSeconds })`.
5. **Verify before acting** with `await okoro.verify(token)` — if `allowed === false`, do not perform the action.

## SDK surface

| Method                          | Purpose                                  |
| ------------------------------- | ---------------------------------------- |
| `okoro.agents.register(input)`  | Register a new agent public key          |
| `okoro.agents.list()`           | List agents owned by this principal      |
| `okoro.agents.show(id)`         | Inspect a single agent                   |
| `okoro.agents.revoke(id)`       | Revoke an agent (irreversible)           |
| `okoro.policies.create(input)`  | Issue a scoped, signed policy            |
| `okoro.policies.list()`         | List active policies                     |
| `okoro.policies.revoke(id)`     | Revoke a policy                          |
| `okoro.verify(token)`           | The verify hot path                      |
| `okoro.audit.list(params)`      | Paginated audit-event reader             |
| `okoro.audit.export(params)`    | NDJSON streaming export for SOC2 evidence |

Both TypeScript and Python SDKs preserve byte-identical JWT shapes — see
the [denial precedence](/docs/concepts/denial-precedence) page for the wire
contract relying parties build retry logic on.
