OKORO is a hot-path service. The verify endpoint sits between AI agents and
the actions they perform, so a regression has customer-visible blast radius.
This page is the canonical entry point for everyone holding the pager.

## Status & SLOs

| Surface             | SLO target            | Reflected in                                   |
| ------------------- | --------------------- | ---------------------------------------------- |

## Deploy & rollback

- **Deploy target**: Railway for the API, Vercel for the dashboard and these docs. See [`docs/DEPLOYMENT_GUIDE.md`](https://github.com/klytics/okoro/blob/main/docs/DEPLOYMENT_GUIDE.md).
- **Migration immutability**: every Prisma migration is append-only after merge. Never edit a previously applied migration in production. Local repair only.
- **Rollback**: code rollbacks are safe; migrations are forward-only. If a migration is the regression, write a forward migration that restores the prior shape.
