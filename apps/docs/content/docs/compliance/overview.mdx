## Audit-chain verification — third-party

Auditors can verify the CERNIQ audit chain offline without any CERNIQ-side
trust:

```ts
import { CerniqVerifier } from '@cerniq/verifier-rp';
import { readFileSync } from 'node:fs';

const verifier = new CerniqVerifier({
  jwksUri: 'https://api.cerniqapp.com/.well-known/audit-signing-key',
});

const ndjson = readFileSync('./audit-export.ndjson', 'utf-8');
const result = await verifier.verifyAuditExport(ndjson);
// { valid: true, count: 12489, brokenAt: null }
```

The verifier checks:
1. Every row's `signature` is valid against the JWKS-published public key (matched by `signingKeyId`).
2. Every row's `prevHash` matches the SHA-256 of the previous row's signed bytes.
3. The first row matches the genesis row pinned in the CERNIQ documentation.

A `false` result identifies the first broken row, so an investigation has a
starting point.

## EU residency

Single-region EU posture available — API, database, and KMS pinned to
`eu-west-1` or operator-specified region. See
[`docs/EU_RESIDENCY.md`](https://github.com/klytics/cerniq/blob/main/docs/EU_RESIDENCY.md).

## Retention

Default 7 years (SOC2 Type II floor; financial-services audit norm).
Operator-configurable per environment.
