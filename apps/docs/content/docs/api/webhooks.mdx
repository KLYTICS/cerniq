---
title: Webhooks API
description: Manage webhook subscriptions. HMAC-signed delivery, retry with backoff, dead-letter queue.
---

See [webhooks concept](/docs/concepts/webhooks) for signature verification,
retry semantics, and the live event catalog.

## Endpoints

| Method | Path                                    | Purpose                                  |
| ------ | --------------------------------------- | ---------------------------------------- |
| POST   | `/v1/webhooks/subscriptions`            | Create a subscription                    |
| GET    | `/v1/webhooks/subscriptions`            | List subscriptions                       |
| GET    | `/v1/webhooks/subscriptions/:id`        | Inspect one                              |
| PATCH  | `/v1/webhooks/subscriptions/:id`        | Update (pause, change events, change URL) |
| DELETE | `/v1/webhooks/subscriptions/:id`        | Delete                                   |
| GET    | `/v1/webhooks/deliveries`               | Recent deliveries (paginated)            |
| GET    | `/v1/webhooks/deliveries/:id`           | Inspect one delivery, including failures |
| POST   | `/v1/webhooks/deliveries/:id/replay`    | Replay a delivery (DLQ recovery)         |

Authentication: `X-CERNIQ-API-Key` with **FULL** scope.

## Create

```bash
curl -X POST https://api.cerniq.io/v1/webhooks/subscriptions \
  -H "X-CERNIQ-API-Key: $CERNIQ_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/cerniq-webhooks",
    "events": [
      "cerniq.agent.trust_score_changed",
      "cerniq.agent.revoked"
    ],
    "active": true
  }'
```

Response includes `signingSecret` (shown **once**). After creation, the
subscription record returns the secret as `signingSecretPrefix` only.
