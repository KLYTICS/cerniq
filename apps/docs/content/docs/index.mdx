CERNIQ is the neutral verification, policy enforcement, and behavioral
attestation layer between AI agents and the services they act on.

- **Public keys only** — CERNIQ never holds, transmits, or generates private keys.
- **Signs what it observed** — every audit row is hash-chained and signed under a published key.
- **Vendor- and model-neutral** — works with any LLM, MCP server, or agent framework.

<SdkVersionBadges />

## The verify loop

1. An agent signs a short-lived JWT proving authority over its identity.
2. The relying party calls `POST /v1/verify` (or runs the offline `@cerniq/verifier-rp`).
3. CERNIQ walks the [denial precedence chain](/docs/concepts/denial-precedence) — first failure wins.
4. A signed audit row is appended to the principal's chain.
