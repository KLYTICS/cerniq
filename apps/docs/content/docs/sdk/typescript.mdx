## Install

```bash
npm install @okoro/sdk
# or
pnpm add @okoro/sdk
# or
yarn add @okoro/sdk
```

## Runtime requirements

- **Node** ≥ 20, or any browser supporting WebCrypto (Chrome 88+, Firefox 87+, Safari 14+), or an edge runtime (Cloudflare Workers, Vercel Edge, Deno).
- **No Node-only imports** — the SDK uses `@noble/ed25519` and Web `fetch`, so it runs identically in browser, edge, and Node contexts.

## Surface at a glance

```ts
import {
  Okoro,            // primary client class
  generateKeypair,  // mint Ed25519 keypair locally (private key never leaves your process)
  sign,             // sign an authority token with a private key
  type DenialReason,
  DENIAL_REASON_PRECEDENCE,
} from '@okoro/sdk';

const okoro = new Okoro({ apiKey: process.env.OKORO_API_KEY! });

await okoro.agents.register({ /* … */ });
await okoro.policies.create({ /* … */ });
await okoro.verify(token);
```

## Common recipes

| Need                                    | Recipe                                                                                                  |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| Verify in a browser without a backend   | Use `@okoro/sdk` directly in the client bundle — it's WebCrypto-only.                                   |
| Verify in an edge function              | Same; no `node:crypto` is imported.                                                                     |
| Verify **offline** (no OKORO round-trip) | Use `@okoro/verifier-rp` instead (see [verifier-rp](/docs/sdk/verifier-rp)).                            |
| Subscribe to lifecycle webhooks         | `okoro.webhooks.subscriptions.create({ url, events })` — see [Webhooks API](/docs/api/webhooks).        |
| Export audit chain for SOC2 evidence    | `okoro.audit.export({ format: 'ndjson' })` — see [Audit API](/docs/api/audit).                          |

## Error handling

Every method throws a typed `OkoroError` subclass on failure. Common shapes:

| Class                | When                                                          |
| -------------------- | ------------------------------------------------------------- |
| `NotFoundError`      | Agent, policy, or audit-event id does not exist               |
| `RateLimitedError`   | Per-tier throttle hit (OD-006); includes `retryAfterSeconds`  |
| `AuthError`          | API key invalid, expired, or insufficient scope               |
| `ValidationError`    | Request body failed Zod validation                            |
| `VerifyDeniedError`  | `verify()` returned `allowed: false`; exposes `denialReason`  |

Build retry / escalation logic on the [denial precedence](/docs/concepts/denial-precedence)
order — not on string equality.
