title: CLI (cerniq)
description: Operator and developer command line. Cobra-based Go binary with kubectl-style plugin discovery.
---

The CERNIQ CLI is a single static Go binary. ~5 MB cross-compiled to
darwin/linux/windows Ă— amd64/arm64. Authenticates via OS keychain
(device-code OAuth on first run, `--api-key` flag for headless / CI).

## Install

```bash
# macOS (Homebrew)
brew install klytics/cerniq/cerniq

# Linux (curl installer)
curl -fsSL https://get.cerniq.dev/install.sh | sh

# Verify
cerniq --version
```

Optionally verify signature with `CERNIQ_VERIFY_SIGNATURE=1` (Sigstore).

## First-run authentication

```bash
cerniq login                  # device-code OAuth (recommended)
cerniq login --api-key $AK    # headless / CI
cerniq whoami
cerniq doctor                 # connectivity, clock skew, JWKS reachability, onboarding state
```

Tokens cached in OS keychain (Keychain.app on darwin, Secret Service on
linux, Credential Manager on windows).

## Command surface

| Namespace            | Commands                                                          |
| -------------------- | ----------------------------------------------------------------- |
| `cerniq init`         | `--industry fintech|ai-platform|saas` â€” scaffold + register first agent |
| `cerniq agents`       | `register|list|show|revoke|status` (`--generate-keypair`)         |
| `cerniq policy`       | `create|list|revoke|inspect` (imperative flags or `--file`)       |
| `cerniq verify`       | Verify a token; renders denial reason in canonical precedence order |
| `cerniq events`       | `list|tail|export` â€” cursor pagination, signal-aware Ctrl-C, NDJSON streaming |
| `cerniq listen`       | `--forward-to <url>` â€” local webhook tunnel (like `stripe listen`) |
| `cerniq trigger`      | Emit a test webhook event for local development                   |
| `cerniq dash`         | TUI cockpit (bubbletea) â€” whoami + last 10 events + last 10 verifies |
| `cerniq kms rotate`   | Rotate the audit signing key                                      |
| `cerniq mcp install`  | Helper: writes Claude Desktop config to enable MCP integration    |
| `cerniq audit *`      | Plugin namespace owned by `cerniq-audit` binary (kubectl-style)    |

Every command supports `--json` for scripted output.

## Plugin discovery (kubectl-style)

Any `cerniq-*` executable on PATH becomes a subcommand:

```bash
# drop `cerniq-hello` somewhere on PATH
$ cerniq hello world
# invokes `cerniq-hello world` with all remaining args forwarded
```

This is how the `cerniq audit` namespace ships â€” the `cerniq-audit` binary
is independent of the core CLI but discovered automatically.

## Config

XDG-compliant: `~/.config/cerniq/config.toml`. API key lives in OS keychain
via [`99designs/keyring`](https://github.com/99designs/keyring). Falls
back to `CERNIQ_API_KEY` env var for CI.
