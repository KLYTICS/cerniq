title: CLI (okoro)
description: Operator and developer command line. Cobra-based Go binary with kubectl-style plugin discovery.
---

The OKORO CLI is a single static Go binary. ~5 MB cross-compiled to
darwin/linux/windows Ă— amd64/arm64. Authenticates via OS keychain
(device-code OAuth on first run, `--api-key` flag for headless / CI).

## Install

```bash
# macOS (Homebrew)
brew install klytics/okoro/okoro

# Linux (curl installer)
curl -fsSL https://get.okoro.dev/install.sh | sh

# Verify
okoro --version
```

Optionally verify signature with `OKORO_VERIFY_SIGNATURE=1` (Sigstore).

## First-run authentication

```bash
okoro login                  # device-code OAuth (recommended)
okoro login --api-key $AK    # headless / CI
okoro whoami
okoro doctor                 # connectivity, clock skew, JWKS reachability, onboarding state
```

Tokens cached in OS keychain (Keychain.app on darwin, Secret Service on
linux, Credential Manager on windows).

## Command surface

| Namespace            | Commands                                                          |
| -------------------- | ----------------------------------------------------------------- |
| `okoro init`         | `--industry fintech|ai-platform|saas` â€” scaffold + register first agent |
| `okoro agents`       | `register|list|show|revoke|status` (`--generate-keypair`)         |
| `okoro policy`       | `create|list|revoke|inspect` (imperative flags or `--file`)       |
| `okoro verify`       | Verify a token; renders denial reason in canonical precedence order |
| `okoro events`       | `list|tail|export` â€” cursor pagination, signal-aware Ctrl-C, NDJSON streaming |
| `okoro listen`       | `--forward-to <url>` â€” local webhook tunnel (like `stripe listen`) |
| `okoro trigger`      | Emit a test webhook event for local development                   |
| `okoro dash`         | TUI cockpit (bubbletea) â€” whoami + last 10 events + last 10 verifies |
| `okoro kms rotate`   | Rotate the audit signing key                                      |
| `okoro mcp install`  | Helper: writes Claude Desktop config to enable MCP integration    |
| `okoro audit *`      | Plugin namespace owned by `okoro-audit` binary (kubectl-style)    |

Every command supports `--json` for scripted output.

## Plugin discovery (kubectl-style)

Any `okoro-*` executable on PATH becomes a subcommand:

```bash
# drop `okoro-hello` somewhere on PATH
$ okoro hello world
# invokes `okoro-hello world` with all remaining args forwarded
```

This is how the `okoro audit` namespace ships â€” the `okoro-audit` binary
is independent of the core CLI but discovered automatically.

## Config

XDG-compliant: `~/.config/okoro/config.toml`. API key lives in OS keychain
via [`99designs/keyring`](https://github.com/99designs/keyring). Falls
back to `OKORO_API_KEY` env var for CI.
