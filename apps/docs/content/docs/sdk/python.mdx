## Runtime requirements

- **Python** ≥ 3.11
- **HTTPS** to `api.okoroapp.com` (or your operator-controlled `OKORO_API_BASE_URL`)

## Surface at a glance

```python
import asyncio
from okoro import AsyncOkoro, generate_keypair, sign

async def main() -> None:
    okoro = AsyncOkoro(api_key="ak_live_...")

    public_key, private_key = generate_keypair()
    agent = await okoro.agents.register(public_key=public_key, label="checkout-bot")

    policy = await okoro.policies.create(
        agent_id=agent.id,
        scope={"actions": ["charge"], "spend_limit_usd": 500},
        ttl_seconds=60,
    )

    token = sign(
        private_key,
        agent_id=agent.id,
        policy_id=policy.id,
        action="charge",
        amount=49,
    )

    result = await okoro.verify(token)
    if not result.allowed:
        raise SystemExit(f"denied: {result.denial_reason}")

asyncio.run(main())
```

## Async-first, sync available

`AsyncOkoro` is the primary client — designed for `asyncio` apps,
FastAPI, LangChain, CrewAI, and async agent frameworks. A thin `Okoro`
synchronous wrapper is also available for scripts and notebooks:

```python
from okoro import Okoro  # sync wrapper

okoro = Okoro(api_key="ak_live_...")
result = okoro.verify(token)
```

## Module map

| Module             | What lives there                                                  |
| ------------------ | ----------------------------------------------------------------- |
| `okoro.agents`     | Register, list, revoke, inspect agent identities                  |
| `okoro.policies`   | Issue, list, revoke scoped policies                               |
| `okoro.verify`     | The verify hot path                                                |
| `okoro.audit`      | Cursor-paginated event reader; NDJSON export                       |
| `okoro.webhooks`   | Subscription + delivery management                                |
| `okoro.crypto`     | `generate_keypair()`, `sign()`, helpers — local-only key material |
| `okoro.errors`     | `OkoroError`, `NotFoundError`, `RateLimitedError`, etc.           |
| `okoro.models`     | Pydantic v2 wire-shape models (`Agent`, `Policy`, `VerifyResult`) |

## Wire parity with `@okoro/sdk` (TypeScript)

Both SDKs produce **byte-equivalent JWT** for the same `(privateKey, claims)`
input. This is enforced by a cross-package parity test at
[`tests/cross-package/sdk-api-jwt-parity.spec.ts`](https://github.com/klytics/okoro/tree/main/tests/cross-package/).
A token minted by the Python SDK can be verified by `@okoro/verifier-rp` and
vice-versa.

## Type safety

Every wire shape is a Pydantic v2 model under `okoro.models`. `mypy
--strict` runs clean in CI. IDEs that support PEP 604 unions get full
autocomplete on `verify()` results.
