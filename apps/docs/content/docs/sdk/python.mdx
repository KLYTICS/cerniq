## Runtime requirements

- **Python** ≥ 3.11
- **HTTPS** to `api.cerniq.io` (or your operator-controlled `CERNIQ_API_BASE_URL`)

## Surface at a glance

```python
import asyncio
from cerniq import AsyncCerniq, generate_keypair, sign

async def main() -> None:
    cerniq = AsyncCerniq(api_key="ak_live_...")

    public_key, private_key = generate_keypair()
    agent = await cerniq.agents.register(public_key=public_key, label="checkout-bot")

    policy = await cerniq.policies.create(
        agent_id=agent.id,
        scope={"actions": ["charge"], "spend_limit_usd": 500},
        ttl_seconds=60,
    )

    token = sign(
        private_key,
        agent_id=agent.id,
        policy_id=policy.id,
        action="charge",
        amount=49,
    )

    result = await cerniq.verify(token)
    if not result.allowed:
        raise SystemExit(f"denied: {result.denial_reason}")

asyncio.run(main())
```

## Async-first, sync available

`AsyncCerniq` is the primary client — designed for `asyncio` apps,
FastAPI, LangChain, CrewAI, and async agent frameworks. A thin `Cerniq`
synchronous wrapper is also available for scripts and notebooks:

```python
from cerniq import Cerniq  # sync wrapper

cerniq = Cerniq(api_key="ak_live_...")
result = cerniq.verify(token)
```

## Module map

| Module             | What lives there                                                  |
| ------------------ | ----------------------------------------------------------------- |
| `cerniq.agents`     | Register, list, revoke, inspect agent identities                  |
| `cerniq.policies`   | Issue, list, revoke scoped policies                               |
| `cerniq.verify`     | The verify hot path                                                |
| `cerniq.audit`      | Cursor-paginated event reader; NDJSON export                       |
| `cerniq.webhooks`   | Subscription + delivery management                                |
| `cerniq.crypto`     | `generate_keypair()`, `sign()`, helpers — local-only key material |
| `cerniq.errors`     | `CerniqError`, `NotFoundError`, `RateLimitedError`, etc.           |
| `cerniq.models`     | Pydantic v2 wire-shape models (`Agent`, `Policy`, `VerifyResult`) |

## Wire parity with `@cerniq/sdk` (TypeScript)

Both SDKs produce **byte-equivalent JWT** for the same `(privateKey, claims)`
input. This is enforced by a cross-package parity test at
[`tests/cross-package/sdk-api-jwt-parity.spec.ts`](https://github.com/klytics/cerniq/tree/main/tests/cross-package/).
A token minted by the Python SDK can be verified by `@cerniq/verifier-rp` and
vice-versa.

## Type safety

Every wire shape is a Pydantic v2 model under `cerniq.models`. `mypy
--strict` runs clean in CI. IDEs that support PEP 604 unions get full
autocomplete on `verify()` results.
